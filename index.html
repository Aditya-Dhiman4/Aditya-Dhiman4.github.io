// Project 01
Formula Electric Device Drivers

Board-level firmware and shared driver architecture underpinning controls, sensing, and power distribution across the car.

Problem
Higher-level vehicle controls are only as reliable as the low-level driver layer that powers sensors, buses, and fault handling.
What I Built
Driver work spanning eFuse power control with DMA-based current sensing, IMU acquisition over SPI, CORDIC hardware acceleration, and CAN-FD messaging under FreeRTOS.
Technical Challenge
Keeping firmware hardware-agnostic across boards while handling asynchronous ADC/DMA timing, retry logic, and tight compute budgets on STM32 MCUs.
Result
3× reduction in trig CPU cost for control math, improved startup robustness, and a cleaner shared firmware base for new board bring-up.

// Project 02
Body Velocity Extended Kalman Filter

Low-speed state estimation for a Formula Electric car where tire slip makes wheel-speed-derived velocity unreliable.

Problem
Launch, braking, and low-speed cornering create operating points where the control stack still needs body velocity, but raw wheel speed is not trustworthy.
What I Built
A low-speed EKF in C++ and Simulink fusing IMU, wheel speed, GPS, and slip ratio gating for cleaner state estimates.
Technical Challenge
Balancing estimator responsiveness against noisy or corrupted inputs while handling nonlinear measurement models and Jacobian derivation under real-time constraints.
Result
Usable velocity estimates below 10 km/h and a stronger foundation for the torque-vectoring and yaw-rate control stack deployed on car.
C++SimulinkEKFGPS / IMU FusionState Estimation
Firmware repo
// Project 03
Yaw Rate Controller & Torque Vectoring

Real-time vehicle-dynamics control work aimed at making torque allocation responsive and stable across the full operating envelope.

Problem
Controlling a Quad Motor over-actuated system requires control logic that stays reliable across load transfer, sensor noise, and hard real-time firmware constraints.
What I Built
A PID-based yaw-rate controller and load-transfer-aware torque allocator modeled in Simulink and deployed into embedded C for the Formula Electric vehicle.
Technical Challenge
Bridging controller design from simulation into deterministic firmware while keeping the loop fast enough for on-car validation without a hardware-in-the-loop rig.
Result
Integrated controller validated alongside the EKF estimator and driver layer; forms the real-time controls backbone currently running on the car.
CSimulinkPIDTorque VectoringVehicle Dynamics
